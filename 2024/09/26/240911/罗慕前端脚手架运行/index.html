罗慕前端脚手架运行

  • 从git上拉下来的前端脚手架有可能不能直接运行，需要做一些处理。

第一步，下载依赖

  • 打开刚刚从 git上拉下来的脚手架，npm run dev 报错，找不到模块，此时是没有下载依赖。

  • 执行命令

第二步，尝试编译，看看有哪些报错

  • 开发要用到 ic 模块，所以我们在config中配置上编译路径，然后尝试编译一下

  • 报错了

从报错上来看，是没有找到相应的文件，这种情况，只需要去目录下看一看有没有那个文件，没有的话，从home–> hotwebs –> nccloud –> resources 中拷贝相应的文件到脚手架即可。（如果你的文件拷过来，重新编译还是报 找不到的错误，重启 vscode试试）

粘贴到脚手架内

此时，重新再变异一下，就没有 scmpub 模块的错误了，剩下的 也是 qc模块 和 so模块的 components 找不到，依法炮制，去home里复制即可。

有四个很重要的模块，如果脚手架里面没有，要去home里复制过来，不管有没有提示报错信息

  • 当上面所有的模块都复制到脚手架中，重新编译，此时已经没有报错了。

第三步，尝试在开发环境启动脚手架

  • 首先启动后端项目

  • 然后启动前端项目

  • 此时是可以进入到主页的（如果主页是空白的，啥都没有，可能是缓存的影响，清理一下即可。）

进入某一节点，页面加载不出来

  • 补上 /nccloud/resources 之后，页面可以正常显示了。
  • 但是，这样子，前端的资源用的是home里的文件，我们希望 用我们脚手架里的文件，那么还是要去掉这个

缺少文件

检查控制台里面报 404的地方，去home把相应的文件夹复制到前端脚手架里面。

编译路径不全

  • 看到这一堆的报错，Uncaught SyntaxError: Cannot use import statement outside a module ， 点击 index.js:2:1 进入到代码界面

看到是 ic/ic/components/模块下的这些组件报的错误，然后我们的工程里面是源码，并不是编译后的代码，那么可能就是在别的地方用到了这些组件，但是脚手架的源码没有编译导致的。

在 config.json中配置好编译路径之后，再重启。
