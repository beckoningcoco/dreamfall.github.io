新建一个全新的NCC项目

这里新建工程的时候，不要大小写都用！后面会出问题的！全用小写！

新建一个NCC组件

新建元数据

  • 在新建的模块 souto 中，METADATA 目录下，新建元数据

  • 切换到 MDP视图可以看到新建的元数据

设置 实体/组件属性面板

  • 画元数据
    • 代码风格 —> 标准格式

设置实体的字段属性以及业务接口映射 （只介绍几个值得注意的地方）

主键

枚举类型

  • 新建完枚举之后，在实体的具体的字段上引用该枚举。

特性
  • 引入需要的特性，一般是下面五个，没有树形结构！

业务流程

  • 引入完，我们的元数据会自动实现一些接口和补充一些字段。

  • 这些自动生成的字段，我们原则上不去修改，让它默认就好，但是名称啊，长度，可见不可见啥的可以修改！
  • code字段 对应 系统里面的单据号
  • id字段要删除掉
业务接口属性映射
  • 新增完特性，以及新增完我们需要的字段属性之后，来到业务接口映射。
  • 首先，所有的字段都要给设置映射属性
  • 第二，id,pid,pk锁，这些地方要设置成实体的主键（我们自己创建的）
  • 第三,单据号 映射为实体的code属性即可

  • 如果你的访问器类型设置成了AggVO，则还需要引入主子表查询接口，这里注意，只有Aggvo支持审批流程！

额外的一个小问题

  • 到了此时，去代码注册里下载源代码的时候，我发现我的菜单选不到。

  • 低代码平台在选择 上级菜单的时候，似乎只能识别到 系统默认的那个菜单模板下的菜单。
  • 所以我们必须将这三条数据复制成 insert语句，改一下 主键和菜单模板的主键

  • 登录 集团管理员 进入 代码定制节点

  • 如果报错 往 sm_appregister 插入数据时，违反了唯一约束XXX ，这个错误也是低代码平台的bug ，在插入数据的时候重复插入两次，第二次爆错了，解决办法， 找到 nc.codeplatform.framework.DataEngine 这个类 在 850 行 加上

  • 再次下载源代码就可以了！

打开代码定制节点报错 授权已过期，等待正版

在补丁文件 modules\lcdp\META-INF\classes\nc\impl\codefactory\codecreater\ICAHandler.class中，有一个时间字段，暂时修改成2026年，如果有需要的话，可以修改成长点的时间，在重新打补丁重启

File Valid ：Invalid directory name

  • 新建工程的时候，工程的名称不要特别复杂，不要 带中文，不要 用特殊符号，不要 大小写字母都用！！！！！
  • 就用简单的 小写字母就行，带几个数字也可以。

  • 因为 你的这个工程名字，是对应元数据的模块名的 —> 生成代码的时候会创建 vo，vo所在的包，包是 包含模块名且没有大写字母的 如果你工程中有大写字母，生成的类，那个类的 包名里面的模块那一层 都是小写的字母，后面校验的时候就会做个对比，就会不一致，然后报错！！
