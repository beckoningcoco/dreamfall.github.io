第四步 编写后端代码

第一步 写一个处理类

  • 写一个处理类 SaleOrderHistoryImportProcess 继承 AbstractImportProcess

  •  package nccloud.so.so.plugin;
import nc.hk.bs.lmsaleorder.revisesave.SaleOrderSingleReviseSaveHandler;
import nc.vo.pub.BusinessException;
import nc.vo.so.m30.revise.entity.SaleOrderHistoryVO;
import nccloud.bs.excel.plugin.AbstractImportProcess;

public class SaleOrderHistoryImportProcess extends AbstractImportProcess {
    private SaleOrderSingleReviseSaveHandler handler;

    @Override
    protected void processBill(Object vo) throws BusinessException {
        try {
            SaleOrderHistoryVO myAggvo = (SaleOrderHistoryVO) vo;
            getHandler().processBatchModify(myAggvo); //我们自己的业务逻辑  校验，保存
        } catch (Exception e) {
            throw new BusinessException(&quot;生成销售订单出错，请检查数据:&quot; + e.getMessage());
        }
    }

    private SaleOrderSingleReviseSaveHandler getHandler() {
        if (handler == null) {
            handler = new SaleOrderSingleReviseSaveHandler();
        }
        return handler;
    }

}

  • ‘nc.vo.so.m30.revise.entity.SaleOrderHistoryVO’ Aggvo全类名 ，自己替换

  • ‘30R’ 单据类型 ，不是交易类型

  • ‘SO’ 模块名

  • ‘000DR300000000SLOD31’ 主键，自己修改一下

  • ‘销售订单修订导入’ 名称 – 随意

  • ‘nccloud.so.so.plugin.SaleOrderHistoryImportProcess’ ， 我们刚才自己新建的 处理类的 全类名

  • ‘2024-11-11 11:00:00’ TS 时间戳

结束 一般到这里 前台点击导入，选择一个模板，就能正常进入到我们的自己的处理类里面，如果不行，可能需要调整 对应的 XML配置文件

配置文件修改，无需重启服务，刷新页面即可。
